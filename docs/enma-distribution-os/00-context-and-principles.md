# Context and Principles

## Context

Enma Labs is moving from website legitimacy work into distribution machinery. The public site now has the Apple-facing basics: privacy, support, contact, terms, legal entity, mailing address, phone, and work email.

The next bottleneck is distribution. Product shipping is becoming easier. Attention, trust, learning loops, and repeated public motion are the harder system.

The distribution system must serve both consumer apps and possible B2B software. It must work across four apps, with Balsam as the first concrete product.

## Operating Shape

The system should be a hybrid machine:

- Agents produce drafts, analysis, experiments, clips, schedules, and reports.
- Humans approve monthly content, major creative direction, ad spend changes, public launches, and sensitive messaging.
- The system learns from results and proposes next moves.
- Nothing public or expensive should mutate silently.

## First-Principles Bottleneck Map

The real system is not just posting or ads. It has ten layers:

- audience identity and memory
- content source of truth
- approvals and governance
- creative production
- social distribution
- paid acquisition and budget optimization
- analytics, attribution, and experiments
- App Store, ASO, and review operations
- community, CRM, and feedback capture
- portfolio tracking across all four apps

## Design Principles

- One Enma system, four app surfaces.
- Source of truth before automation.
- Measurement before scale.
- Approval before public mutation.
- Open-source first, SaaS only when the open-source option creates more drag than leverage.
- Agent control through MCP where native support exists, API wrappers where it does not.
- Content should carry the Enma voice, but operational tools should stay plain and auditable.

## What This System Is Not

- It is not a pile of tools.
- It is not a social scheduler with extra dashboards.
- It is not a fully autonomous publisher.
- It is not a replacement for taste, product judgment, or founder voice.
- It is not four separate growth stacks.
