# 03. Narrative Sequence

The homepage should move in this order:

1. Open the room.
2. Name the refusal.
3. Show the first instrument.
4. Let the visitor feel how it behaves.
5. Deepen trust with the journal.
6. Leave one quiet invitation.

Section jobs:

## Hero
- emotional opening
- one thesis
- one clear primary action

## Manifesto
- name what this software refuses to become
- explain the difference without preaching

## Work
- foreground Balsam
- keep future work secondary

## Balsam Preview
- the first product proof surface
- should feel like a believable study of use
- should show behavior, not feature theater

## Journal
- reinforces authorship and continuity
- should read like public notes from the making

## Closing
- calm ending
- one specific ask
- no spectacle for its own sake

Narrative anti-patterns:

- manifesto before product for too long
- future products weighted equally with the only real one
- journal as mood board instead of authored notes
- closing CTA stronger in style than in truth
