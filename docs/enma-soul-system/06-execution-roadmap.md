# 06. Execution Roadmap

This is the sequence for future iterations.

## Phase 1. Lock the upstream system
- keep this folder current
- keep `docs/CREATIVE-DIRECTION.md` aligned
- reduce drift between CSS, theme, and design-system layers

## Phase 2. Make Balsam unmistakable
- tighten hero clarity without flattening the voice
- improve the Balsam preview until it feels like product truth
- keep future lab material secondary

## Phase 3. Upgrade the hero image
- generate 3 controlled candidates
- review them against the brief in `02-image-direction.md`
- reject anything that feels generic or over-rendered

## Phase 4. Raise the interaction floor
- verify mobile nav
- improve waitlist accessibility
- audit motion/reduced-motion behavior
- check keyboard traversal and screen-reader naming

## Phase 5. Earn richer proof
- real Balsam screens
- real beta updates
- real trust signals
- more product-specific journal notes

Shipping rule:
- do not push production from a round that only made the site prettier
- push when the round made the site truer, clearer, and more coherent
